# 水母陪伴 App · 产品提案与 PRD

## 一、一句话定义

一只住在你电脑屏幕边缘的小水母，在你上班、专注、休息、发呆的时候，用很轻的方式陪着你。

它不是效率工具，也不是养成游戏。它是一个**低打扰的桌面情绪陪伴生物**——核心关键词：低打扰、治愈、陪伴感、轻互动、有边界感。

---

## 二、要解决的问题

打工人每天独自面对电脑 8 小时以上。市面上的产品要么是效率工具（番茄钟、待办），要么是需要照顾的电子宠物（喂养、打卡、升级）。前者冰冷，后者制造新的负担。

中间存在一个空白：**一个不需要被照顾、却让人感到"不是一个人在上班"的陪伴存在。**

水母陪伴 App 填的就是这个空白——它只是"在"，不索取，不评判，不催促。

---

## 三、目标用户

**第一目标用户：上班族打工人（桌面陪伴 / "上班搭子"场景）**

- 画像：每天长时间对着电脑工作的白领、远程办公者、自由职业者。
- 核心诉求：工作时长时间独处，需要一点"有人陪着"的轻松感，但不希望被打扰、不希望多一件要操心的事。
- 为什么选这个人群：使用场景明确（电脑桌面）、付费能力较强、对"陪伴感"有真实需求。

**延伸用户（PRD 记录，暂不作为首发重点）**

- 学生：学习陪伴、专注、作息。
- 情绪压力人群：深夜、焦虑时段的陪伴。

---

## 四、产品定位与差异点

| 对比对象 | 它们是什么 | 本产品的差异 |
|---|---|---|
| 番茄钟 / 待办类 | 效率工具，功能导向 | 不以效率为目标，以陪伴感为目标 |
| 桌面宠物 / 养成类 | 需要喂养、打卡、升级 | 零养成负担，不需要被照顾 |
| AI 聊天陪伴 | 大段文字对话，易显假 | 行为共情为主、轻文本为辅，不假装懂全部情绪 |

**最强记忆点：** 水母从屏幕边缘"探出身子"。它不是平铺在一个窗口里，而像真的住在你屏幕的边缘——这一个动作构成产品的 IP 识别度。

**核心设计红线（产品最该避免的）：**
1. 不做成普通桌宠（只会动一动、升级）。
2. 不做成 AI 心理咨询（不过度安慰、不假装全懂）。
3. 不堆任务系统（不做金币、签到、商店、升级链）。
4. 不打扰用户（陪伴的本质是"在"，不是"烦"）——必须有"让它退下"的能力。

## 五、核心场景

### 场景 1 · 静默陪伴（默认态）
用户正常工作，水母在屏幕边缘安静悬浮，只露出半个身子，保持微微呼吸般的动态。
表达："我不打扰你，但我在。"

长时间（60 秒）无任何页面操作后，水母自动进入**半透明陪伴**——仍在，但更隐形；任意点击、鼠标移动或按键即恢复可见。

### 场景 2 · 手势互动
用户通过轻量手势与水母互动，动作始终柔软克制：

| 手势 | 行为 |
|---|---|
| 单击 | 从边缘探出、吐短泡泡打招呼 |
| 双击 | 开启 / 退出 25 分钟专注陪伴 |
| 悬停 3s | 进入好奇状态，可展开轻量对话 |
| 拖拽 | 换位置，贴边自动吸附 |
| 三击 | 手动进入半透明（单击恢复） |
| 连点 4 次 | 被戳反应（趣味彩蛋） |
| 长按 0.8s | 打开设置面板 |

文本必须克制，越轻越好——行为共情为主，短句为辅。

### 场景 3 · 情绪状态回应
水母感知用户的行为节奏，切换状态：

- **开启专注** → 暖黄发光，屏幕角标计时（"我陪你待着"）
- **完成专注** → 开心态，转圈 + 短句鼓励
- **深夜还在（23:00–08:00，本地时区）** → 困倦态，颜色变柔、动作变慢
- **回到桌面** → 欢迎回来；2 分钟内频繁回来则静默出现，不再反复转圈
- **鼠标靠近** → 眼球追踪 + 好奇态

核心原则：感知状态，柔软回应，**绝不评判、不催促**。

### 场景 4 · 让它退下（边界感场景）★ 本产品理念的关键证明
用户需要全屏演示、看视频或暂时不需要陪伴时，可让水母"退下"——游出屏幕，边缘留召回入口（🪼）。也可手动半透明或拖拽换位置。

这个场景是产品"低打扰、有边界感"理念的直接体现，也是它区别于普通桌宠的关键。

### 场景 5 · 新手引导
首次使用时，8 步交互式教学（靠近 → 单击 → 双击 → 拖拽 → 连点 → 长按），像游戏新手教程，30 秒内学会与水母互动。

## 六、水母的基础状态（MVP 范围）

不做复杂养成。第一版以 5 种核心状态为主，另加 1 个趣味彩蛋态。

| 状态 | 触发 | 表现 | 传达 |
|---|---|---|---|
| 安静 | 默认 / 早 8 点恢复 | 边缘悬浮、微微呼吸 | 我在 |
| 好奇 | 靠近 / 悬停 3s / 被戳后 | 探出身子、蓝色光晕 | 你找我吗 |
| 陪伴·专注 | 双击开启专注 | 暖黄发光、计时角标 | 我陪你待着 |
| 困倦 | 23:00–08:00 本地时区 | 紫色、动作变慢、画面略暗 | 该休息啦 |
| 开心 | 完成专注 / 回到桌面 | 粉色、转圈、吐泡泡 | 你做到了 |
| 被戳（彩蛋） | 连点 4 次 | 橙色、摇晃、弹开眼 | 别戳了啦 |

**好奇态对话卡（轻量，非 AI 长聊）：**
用户在高好奇态下单击，弹出预设选项——「今天有点累」「心情还好」「有点压力」「陪我专注」——点选后切换对应情绪 + 一句短回应。

## 七、MVP 功能清单

第一版只验证一个核心假设：**"一只桌面水母的陪伴感是否成立"。** 范围必须克制。

1. 桌面边缘常驻的水母，有自然呼吸般动态。
2. 单击探出 / 游出；拖拽换位置、贴边吸附。
3. 五种核心情绪状态 + 被戳彩蛋态，及状态间切换。
4. 极简泡泡文案（轻共情，非鸡汤）。
5. 边界控制——退下 / 半透明 / 调整位置 / 边缘召回。
6. 与专注计时联动（双击 25 分钟，结束进入开心态）。
7. 深夜困倦（23:00–08:00 自动进入，08:00 自动恢复安静）。
8. 空闲半透明（60 秒无操作自动半透明，任意操作恢复）。
9. 新手引导（8 步交互教学）。

**明确不做（第一版）：** 养成升级、金币商店、AI 长对话、复杂数据报表。

## 八、原型与正式产品的差异说明

当前 Web 原型（v2）用于验证体验，以下能力在原型中为**模拟**，正式 App 需系统级实现：

| 能力 | 原型实现 | 正式 App |
|---|---|---|
| 桌面常驻 / 透明悬浮 | 浏览器全屏模拟 | macOS 原生透明窗口（NSPanel） |
| 离开 / 回来感知 | 鼠标离开窗口、标签页切换 | 系统前台 / 空闲状态感知 |
| 深夜困倦 | 本地时区 23:00–08:00 | 同上 + 可配置 |
| 设置面板 | 长按打开 + 调试面板手动模拟 | 系统感知自动触发，无调试入口 |
| 半透明 / 退下 | 完整可交互 | 同上 |

## 九、商业化方向：订阅制

采用**订阅制**：基础陪伴功能免费，订阅解锁"更多陪伴功能"。

订阅可解锁的方向（示例）：
- 更多水母品种与情绪皮肤（夜光、季节限定等）。
- 更多水域主题背景与氛围音。
- 更丰富的情绪回应与泡泡文案库。
- 多设备同步陪伴状态。

选择订阅制而非买断的理由：陪伴是持续性体验，订阅模式与"长期相伴"的产品气质一致；也便于持续更新内容维持新鲜感。

## 十、技术路线说明 ★ 重要

**产品终局形态：桌面原生 App（优先 macOS），后续扩展 Windows / 移动端。**

理由：本产品最强的体验——屏幕边缘探出身子、常驻悬浮、透明穿透、系统空闲感知——依赖操作系统级权限（悬浮窗、Menu Bar、前台状态）。这些是网页无法实现的，因此产品要完整落地，终局必须是原生 App。

**当前阶段交付物：Web 桌面端高保真交互原型 v2。**

- 本地文件：`index.html`（部署入口）、`jellyfish-user-prototype-v2.html`（源文件）
- 在线预览：https://1ivy403.github.io/jellyfish-prototype/
- 技术栈：HTML / CSS / JavaScript 单文件
- 目标：最低成本验证"陪伴感"假设，可演示、可分享、可用户测试

**推荐正式开发方案：**
- macOS 单独做 → **Swift + SwiftUI/AppKit**（NSPanel 透明窗口 + WKWebView 或 SwiftUI 重绘）
- 需跨平台 → **Tauri 2** + 现有 Web 交互层

**这一阶段对外可如实表述为：** "终局是原生桌面 App；当前 Web 原型用于验证核心交互与陪伴感假设。"
